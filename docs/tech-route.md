# 技术路线（建议）

## 关键定位（先统一认知）

Aiulink **本身就是企业系统**，不是“先接一堆第三方系统”的中间层。
核心链路应先做：
1. 企业数据导入与治理
2. 企业知识索引与权限隔离
3. 主干 AI 基于企业数据做决策与分发

## 阶段 0：MVP（4-6 周）

目标：跑通“自然语言建代理 + 数据导入 + 低风险自动化 + 人工兜底”。

- 后端：FastAPI + Postgres（元数据）
- 数据层：
  - 本地数据接入（文件夹/CSV/Excel/PDF/图片）
  - 云存储接入（对象存储桶）
  - OCR 与结构化抽取
- 模型层：
  - 小模型做分类、路由
  - 大模型做复杂理解与总结
- 前端：Next.js + React Flow（网络可视化）+ 悬浮控制框
- 鉴权：OIDC（组织、角色、节点身份）

验收指标：
- 自然语言创建代理成功率 > 80%
- 数据导入成功率 > 95%
- 低风险任务自动完成率 > 70%
- 高风险任务 100% 经过人工确认

## 阶段 1：企业数据底座（6-10 周）

- 数据导入中心：批量导入、增量同步、版本快照
- 数据目录与血缘：数据集、字段、来源、更新时间
- 权限切片：按部门/项目/字段控制代理可见性
- 向量索引与检索：让主干 AI 的决策有企业数据支撑
- 审计中心：链路日志、证据快照、责任归因

## 阶段 2：规模化（10-16 周）

- 事件驱动编排（Kafka + Temporal）
- 多租户隔离
- 成本与配额系统
- 异常演练与回滚

## 阶段 3：生态扩展（16+ 周）

- 代理模板市场（按部门/场景复用）
- 组织知识蒸馏（高绩效流程沉淀）
- 合规能力（敏感数据、跨境、行业监管）
- 第三方系统连接器（仅在业务需要时按需接入）

## 关键技术选型说明

1. **FastAPI**：快速构建控制面 API 与策略服务。
2. **Postgres + Object Storage**：承载企业结构化与非结构化数据。
3. **OCR + ETL Pipeline**：把图片/PDF/表格转化为可检索可推理数据。
4. **Temporal/Celery**：支持长任务与失败补偿。
5. **OPA**：把“全代理模式仅限低风险任务”落地为可执行策略。
6. **OpenTelemetry**：实现跨代理链路追踪与责任回放。
