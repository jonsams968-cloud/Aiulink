# 技术路线（建议）

## 阶段 0：MVP（4-6 周）

目标：跑通“自然语言建代理 + 低风险自动化 + 人工兜底”。

- 后端：FastAPI + Pydantic + Postgres
- 任务队列：Celery + Redis（先快后稳）
- 模型层：
  - 小模型做分类、路由
  - 大模型做复杂理解与总结
- 前端：Next.js + React Flow（网络可视化）+ 悬浮控制框
- 鉴权：Keycloak / Auth0（OIDC）

验收指标：
- 自然语言创建代理成功率 > 80%
- 低风险任务自动完成率 > 70%
- 高风险任务 100% 经过人工确认

## 阶段 1：企业集成（6-10 周）

- 接入企业系统：飞书/企微、表格系统、文档系统、工单系统
- 引入能力市场：代理能力发布、订阅、评分与回收
- 策略中心：基于 OPA 的风险与责任策略
- 审计中心：链路日志、证据快照、责任归因

## 阶段 2：规模化（10-16 周）

- 事件驱动编排（Kafka + Temporal）
- 多租户隔离
- 成本与配额系统
- 异常演练与回滚

## 阶段 3：平台化（16+ 周）

- 代理模板市场（按部门/场景复用）
- 组织知识蒸馏（高绩效流程沉淀）
- 合规能力（敏感数据、跨境、行业监管）

## 关键技术选型说明

1. **FastAPI**：快速构建控制面 API 与策略服务。
2. **Pydantic**：保证代理定义、任务计划、审计事件的结构化。
3. **Celery/Temporal**：支持长任务与失败补偿。
4. **OPA**：把“全代理模式仅限低风险任务”落地为可执行策略。
5. **OpenTelemetry**：实现跨代理链路追踪与责任回放。
